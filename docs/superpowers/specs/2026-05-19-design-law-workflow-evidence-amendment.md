# Design Law Amendment For Workflow Evidence Surfaces

## Purpose

This spec defines how `DESIGN.md` should change so Clearpage can support modern workflow evidence surfaces without losing its product character.

The problem is not that the current design law values calm, restraint, or truth.

The problem is that parts of the current law overextend homepage grammar into route classes that need more compositional freedom below the first fold.

`/batch` exposed that problem clearly.

## The Core Diagnosis

The current constitutional layer is too rigid in three places:

1. it treats first-fold discipline as if it should govern the entire scroll length of every route
2. it over-polices workflow and evidence composition by banning too many valid section types
3. it still contains stale instructions that no longer match the shipped product, including the font rule

## What Must Stay

These parts of the current design law are still correct and should remain:

- trust over spectacle
- one dominant working surface on first view
- no gradients
- no glow
- no decorative glass or blur
- restrained motion
- calm token system
- route-class thinking
- no fake feature buffet on the homepage
- no decorative fragmentation before the first task is clear

The redesign problem is not caused by these rules.

## What Must Change

### 1. The font rule must be rewritten

The current law still says not to change the fonts and still reflects older font assumptions.

That is obsolete.

The law must instead define the approved current font constitution explicitly:

- `Newsreader` for display headings
- `Geist Sans` for UI and body text

Future changes should require an explicit constitutional update, but the law must describe current truth.

### 2. Vertical centering must be scoped

The current vertical-centering rule belongs to core tool surfaces like the homepage first view.

It must not apply as a page-wide rule for long workflow or evidence surfaces.

The amended law should say:

- core tool surfaces may use centered viewport composition
- workflow surfaces may anchor to the top once the route contains meaningful below-fold evidence or state depth

### 3. Functional pages need a legal below-fold evidence layer

The current law is too broad when it bans `marketing copy or "product features" sections on functional pages`.

That wording collapses two different things into one bucket.

The amendment should distinguish between:

- decorative marketing sections that dilute the route's job
- evidence layers that directly prove the route's value, limits, and trust model

Functional pages should be allowed to have the second and still ban the first.

### 4. Component law must stop freezing the system

The current instruction to use only pre-described components is too restrictive for evolving product surfaces.

That rule encourages repeated safe utility panels instead of route-specific composition.

The amended law should allow new component types when:

- they serve the route's job directly
- they respect the token system
- they remain low-spectacle
- they are documented in the design law once approved

### 5. Workflow evidence surfaces need their own section

`DESIGN.md` should gain a new route-class subsection for workflow evidence surfaces.

This is not a new top-level product route class. It is a compositional allowance inside workflow pages that need below-fold proof.

Examples include:

- `/batch` below the first fold
- future diagnostics or methodology layers attached to workflow routes

This section should define what is allowed below the fold when a workflow page needs to prove trust.

## New Workflow Evidence Rules

### Rule 1: First-fold discipline remains intact

The first fold still needs one dominant task surface.

Evidence layers must begin below that task surface.

### Rule 2: Proof artifacts are preferred over prose mass

If a trust claim can be shown through a product crop, comparison block, status ladder, process strip, or other structured artifact, that artifact should carry the teaching burden before a long paragraph does.

### Rule 3: Strong section contrast is allowed

Workflow evidence sections may use different internal grammars from section to section.

Allowed differences include:

- screenshot-led sections
- comparison-led sections
- process-led sections
- compact question blocks
- asymmetrical layouts
- restrained section banding or inset framing

This flexibility is allowed only below the fold and only when it serves comprehension.

### Rule 4: Repetition is a smell

If every section uses the same card stack, the same paragraph rhythm, or the same container logic, the composition is likely too flat.

Workflow evidence pages should vary artifact type and density according to the burden of the section.

### Rule 5: Real product evidence is preferred

When a route already has a meaningful product surface, below-fold proof should use captured product evidence where possible.

Real route screenshots or crops are preferred over invented faux-product mockups.

### Rule 6: FAQ is subordinate

FAQ may appear on workflow evidence surfaces, but it should stay low-weight and late in the page.

It should answer objections after proof has already been shown.

### Rule 7: Calm does not mean uniform

Calm means restrained color, material, and motion.

It does not require every section to use the same layout or the same container treatment.

## Wording Changes Needed In `DESIGN.md`

The following current ideas should be narrowed or rewritten:

### Replace

`Do not change the fonts.`

### With

`Use the current approved Clearpage font system. Any future font change requires an explicit design-law amendment.`

### Replace

`Every primary page layout must vertically and horizontally centre the main content group within the full viewport height.`

### With

`Core tool first views may use centered viewport composition. Workflow pages with meaningful below-fold evidence or state depth may anchor the page to the top after the first working surface.`

### Replace

`Marketing copy or "product features" sections on functional pages` as a blanket forbidden item.

### With

`Decorative marketing sections that weaken a functional page's job are forbidden. Evidence layers that directly prove the route's value, trust model, limits, or recoverability are allowed below the first working surface.`

### Replace

`Use only components described in Section 5.`

### With

`Prefer documented components first. New component types are allowed when a route requires them, provided they preserve the token system, restraint model, and route purpose, and are documented in the design law once approved.`

## What This Amendment Prevents

This amendment is meant to prevent three recurring failures:

- homepage rules being copied onto every route without regard for route burden
- calm design collapsing into weak and repetitive composition
- long-form workflow pages defaulting to prose plus polite cards because the law left no better legal option

## Success Criteria

This amendment is successful when:

- core tool surfaces remain highly disciplined
- workflow routes can carry proof-heavy below-fold sections without violating the design system
- future route work can be modern without slipping into gradient-heavy or AI-gimmick styling
- the law reflects shipped product truth instead of stale assumptions

## Out Of Scope

This amendment does not authorize:

- gradients
- glow
- glassmorphism
- dark mode by default
- decorative animation
- generic marketing-page cloning
- abandoning the calm Clearpage token system
